=========================
Stitch design.md Overview
=========================

This document provides a summary of the ``design.md`` format used by Google Stitch for AI-assisted design. The ``design.md`` file serves as a structured Markdown specification that guides the UI generation process.

Core Principles
===============

* **Markdown format:** The file must be written in standard Markdown, utilizing headers, lists, and code blocks appropriately.
* **Declarative design:** Describe *what* the UI should look like and how it behaves, rather than *how* to implement it in code.
* **Structured sections:** A well-formed ``design.md`` typically includes specific sections to provide context, constraints, and component details to the AI.

Typical Structure
=================

A standard ``design.md`` file should include the following sections:

1. **Title and Overview:** A clear heading describing the UI component or page, followed by a brief summary of its purpose.
2. **Key Requirements / Features:** A bulleted list of essential functionalities and elements that must be present in the UI.
3. **Visual Style / Theming (Optional):** Instructions regarding color palettes, typography, and general aesthetic guidelines.
4. **Layout / Structure:** A description of how elements should be arranged on the screen (e.g., header, main content area, sidebar, footer).
5. **Component Details:** Specific breakdowns of individual components, including their states (hover, active, disabled) and data bindings.
6. **Interactions:** Descriptions of how the user interacts with the UI and what the expected outcomes are.

Writing Guidelines
==================

* Be as specific as possible about layout and visual hierarchy.
* Use terminology familiar to designers and front-end developers (e.g., "flexbox", "padding", "primary button").
* Provide example data or content if it helps clarify the layout's intent.
* If referencing existing components in a design system, explicitly name them.

.. note::
   Remember that the ultimate output is UI code generated by Stitch. The clearer and more structured your ``design.md`` is, the better the generated result will be.
