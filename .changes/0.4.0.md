## 0.4.0 - 2026-04-19
### Added
* Go-first CLI/MCP unification: asctl Go binary replaces Python ref CLI, pi-mcp MCP server wraps pi-rpc session service, goreleaser multi-binary releases, go.work workspace, lefthook unified hooks, per-skill pixi environments for Python skills (csv/docx/xlsx/pdf), and ty type checking.
### Changed
* Restructured CONTRIBUTING.md into five sections (Quick Start, Repo Layout, Development Workflows, Validation & Testing, Submitting Changes) covering the new asctl CLI, MCP server pattern, per-skill pixi environments, TDD discipline, and the goreleaser-driven release flow.
### Removed
* Remove stale bootstrap artifacts: pi-rpc Makefile `setup-mcp-dep` target and the corresponding `go get` comment at the top of cmd/pi-mcp/main.go. The mcp-go dependency is declared in go.mod; the setup instructions are no longer needed.
### Fixed
* Fixes GetMessages returning only timestampMs — now parses message_end events (#66)
* Upgrades changie 20-word checklist item to active self-check and adds Trim Example section (#67)
* Fixes pi-rpc passing --thinking-level to pi subprocess — correct flag is --thinking (#65)
* Adds Rule #1 fusion guard to changie Validation Checklist and fusion Bad/Good example (#64)
* PR #69 review fixes for the go-first dev loop. Qualified lefthook pixi tasks with `-e default` to resolve multi-environment task ambiguity, enforced a changie fragment on pre-push, and added per-skill pytest jobs for the csv/docx/xlsx/pdf environments. Removed duplicate findSkillMD in tools/asctl/internal/validator (now calls parser.FindSkillMD) and added errcheck/gosec/unused to golangci-lint. Added [tool.ruff] configuration and excluded src/skills/ref/** from ty's legacy path. The .devcontainer now installs Go 1.25, lefthook, and changie; a postCreate setup.sh symlinks all skills/* into ~/.claude/skills, runs pixi install + lefthook install, and pre-builds asctl into ~/.local/bin so the Claude CLI sees every skill on first boot.
* Add cross-compile target to pi-rpc Makefile producing pi-server-<os>-<arch> binaries for linux/{amd64,arm64} and darwin/{amd64,arm64}; required by agent-extensions build-pi-server.yml
* pi-cli `session create` test coverage — add positive-case `TestRunSessionCreateDefaultsAccepted` asserting empty provider/model are forwarded verbatim so the server can apply PI_DEFAULT_PROVIDER / PI_DEFAULT_MODEL (closes
* pi-rpc cross-compile target now exits non-zero on first failed arch build (PR #69 review)
* asctl frontmatter parse now matches `---` only on line boundaries so body content cannot truncate it (PR #69 review)
* asctl parser treats `name: ~` or empty `name:` as missing instead of literal `<nil>` (PR #69 review)
### Security
* Harden the pi-mcp / asctl trust boundary to the host LLM. pi-mcp now maps ConnectRPC errors to a fixed set of sanitised messages (no paths, tokens, or upstream error text cross the stdio wall) and logs the raw error via slog for local operators. asctl's <available_skills> generator now escapes every text node with encoding/xml.EscapeText (replacing html.EscapeString) so skill metadata and filesystem-controlled <location> paths can never inject XML control characters into the system prompt. Untrusted MCP numeric inputs are clamped to int32 range to prevent protobuf overflow.
