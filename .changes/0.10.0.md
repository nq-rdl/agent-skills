## 0.10.0 - 2026-06-21
### Added
* cc-web-setup: new skill that bootstraps a repository for Claude Code on the web — installs a parameterized `.claude/settings.json` plus portable `web-bootstrap.sh` (SessionStart hook), `cc-web-setup.sh` (pre-snapshot setup script that pre-seeds the RDL marketplace), and `announce-capabilities.sh`. Project-specific dependencies stay in an optional `*.local.sh` seam. Ships unit + integration tests (gh/codex provisioning, parameterized plugin pre-seed, CLAUDE_CODE_REMOTE gate).
* devcontainer: add a static, Docker-free TDD test suite (tests/test_devcontainer.py) that locks the .devcontainer contract — config wiring (postCreate/postStart/waitFor, NET_ADMIN, remoteUser, mounts, claude-code extension), a fail-closed firewall with the required egress allowlist, Dockerfile/.go-version coherence, least-privilege sudoers, and bash -n on the shell scripts.
### Fixed
* cc-web-setup: second review pass — ensure_codex_auth() now trims surrounding whitespace from CODEX_ACCESS_TOKEN before piping it to `codex login --with-access-token` (a secret read from a file commonly carries a trailing newline, which codex would otherwise reject as a malformed JWT), and when the raw value carried whitespace it then drops the malformed live token so later `codex exec` calls fall back to the freshly written auth.json instead of re-parsing it; the `shellcheck-cc-web-setup` lefthook hook no longer ends in `|| true`, so it skips silently when shellcheck is absent but a real lint failure now blocks the commit instead of being swallowed; and the devcontainer `test_firewall_self_verification` test asserts the actual fail-closed control flow (reaching example.com exits 1, unreachable api.github.com exits 1) — constrained to the `then` branch so moving `exit 1` into `else` no longer satisfies it — rather than a substring check that passed even when the verification was inverted or removed.
* cc-web-setup: review hardening — write_stub() in the bootstrap test harness now removes any pre-existing entry before writing so the redirect cannot follow a coreutils symlink onto the real system binary; announce-capabilities.sh's YAML description extractor now stops a block/folded scalar at a column-0 line (the closing `---` or a sibling key) and yields an empty description instead of leaking the next key or a bare `>-`/`|` indicator; and the devcontainer Dockerfile install test asserts apt packages within the apt-get install block and the npm/installer tools against their actual install line (not a comment) rather than a false-positive-prone substring check; and ensure_codex_auth() now actually drops a whitespace-only CODEX_ACCESS_TOKEN — unsetting it in-process and persisting `unset CODEX_ACCESS_TOKEN` to CLAUDE_ENV_FILE (as the auth.json-blob path already did) so codex cannot parse the blank value as a malformed JWT at runtime.
